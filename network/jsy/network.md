## :nerd_face:	What I study
### 1. 용어 정리
#### 1) 네트워크
- 여러 대의 컴퓨터를 서로 연결하여, 서로 데이터를 주고받을 수 있도록 망이 사전에 구축된 것
- 컴퓨터와 같은 노드들이 통신 기술을 이용하여 그물망처럼 연결된 통신 이용 형태
  - 노드 예시) 컴퓨터, 프린터, 스마트폰, 노트북, 데이터 송수신 장치 등등
- 2대 이상의 컴퓨터들을 연결하고 서로 통신할 수 있는 것

#### 2) 인터넷
- 가장 규모가 큰 네트워크
- 인터넷을 통해 서로 다른 네트워크에 속해 있더라도 데이터를 주고 받을 수 있다.
- 인터넷은 'client'와 'server'로 구성되어 있으며, 'TCP/IP'라는 기본 프로토콜을 통해 제공된다.

#### 3) 프로토콜
- 서로 다른 시스템 및 기기 간 데이터 교환을 원활히 하기 위해 표준화된 통신규약
- 기기 간 통신은 교환되는 데이터 형식에 대해 상호 협의를 요구한다.
- 그러므로, 공통의 데이터 교환 방법이나 순서에 대한 약속, 규칙 체계가 필요하다.
- 이런 약속, 규칙을 프로토콜이라고 부른다.
- 동일한 프로토콜을 사용하면 기기 종류 or 프로그램이 다르더라도 컴퓨터간 상호 통신이 가능하다.

#### 4) MAC 주소
- 네트워크 상에서 서로를 구분하기 위해 기기마다 고유하게 할당된 물리적 주소
- 컴퓨터에 장착된 LAN 카드를 구별하기 위해 만들어진 식별 번호
- 통신을 하기 위해 꼭 알아야 하는 주소
<br><br/>

### 2. 네트워크 통신방식
#### 1) 유니캐스트 (Unicast)
- 1:1 통신
- 원하는 대상 하나를 정해서 그 대상과 통신
- LAN 통신에서 송신자의 MAC 주소와 수신자 MAC 주소를 알 때 통신한다.
  - 출발지와 목적지가 정확해야 한다.
- CPU 성능에는 영향을 주지 않는다.
- 대량으로 배포되는 특정 네트워크 응용 프로그램에서 데이터를 전송할 경우, 각각의 네트워크 연결마다 호스트의 컴퓨팅 리소스를 소비할 뿐 아니라 각각 다른 네트워크 대역폭을 필요로 하기 때문에 전송 비용이 매우 높다.
#### 2) 브로드캐스트 (Broadcast)
- 1:ALL 통신
- 내 의지와 상관없이 무조건 받아들여야 하는 통신
- 네트워크 상의 모든 장치가 수신 할 패킷을 전송하는 것
- 언제 사용하는 통신방식일까? <br> 1)송신자는 확실하나, 수신자가 특정하지 않을 때 - 수신자의 IP는 알지만, MAC을 모르는 경우 <br> 2)네트워크에 있는 모든 서버에게 정보를 알려야 할 때 <br> 3)라우터끼리 정보를 교환하거나, 새로운 라우터를 찾을 때
- 네트워크에 있는 모든 목적지에 패킷이 전송되므로 트래픽 증가 및 CPU 성능의 저하에 영향을 준다.
#### 3) 멀티캐스트 (Multicast)
- 1:N 통신
- 원하는 대상 여러개를 정해서 통신
- 수신자를 그룹화하여, 해당 그룹에 해당하는 수신자만 unicast + broadcast 한다.
  - 왜 unicast? 그룹으로 특정하기 때문
  - 왜 broadcast? 그룹 안에 있는 모든 수신지에 정보를 전달하기 때문
#### 4) 애니캐스트 (Anycast)
- 가장 가까운 노드와 통신
- 송신자가 네트워크에 연결된 수신 가능한 수신자 중에서 한 수신자에게만 데이터를 전송한다. (unicast와 다른 점)
- Routing Protocol의 알고리즘에 따라 가장 가까운 노드를 판단하여 데이터를 전송한다.
<br><br/>

### 3. TCP/IP 모델
- TCP, IP 프로토콜을 OSI 7계층 형식에 맞추어 더 추상화시킨 모델
- 4개의 계층으로 구성
#### 1) 응용 계층
- 사용자가 실제로 체감할 수 있는 서비스를 제공하는 계층
- 응용 프로그램 간의 데이터 송수신을 제공한다.
#### 2) 전송 계층
- 응용 계층의 프로그램에서 전달받은 데이터를 목적지 응용 계층의 프로그램까지 전달한다.
- 호스트들 간의 신뢰성 있는 통신을 제공한다. (데이터가 제대로 전달되지 않았을 때, 재전송한다.)
#### 3) 인터넷 계층
- 데이터 주소 정보를 덧붙여 목적지까지 무사히 전달하는 역할을 한다.
- 데이터 전송을 위한 주소, 경로 설정을 제공한다.
- 경로를 설정하기 위해, 라우터라는 장비를 사용한다.
#### 4) 네트워크 인터페이스 계층
- 유선(무선) LAN 어댑터가 처리할 수 있는 형태로 데이터를 변환한다.
- 변환된 데이터를 목적지까지 전달한다.
- 실제 데이터를 송수신한다.
<br><br/>

### 4. OSI 7계층 모델
- 컴퓨터와 컴퓨터 사이의 데이터 전송을 분류한 모델
- 통신 접속부터 완료까지의 과정을 계층화하여 정의한 국제 통신 표준 모델
- 7개의 계층으로 구성
#### 1) 응용 계층
- 사용자가 네트워크에 접근할 수 있도록 해주는 계층
- ex. FTP, HTTP 등
#### 2) 표현 계층
- 서로 다른 어플리케이션들이 서로의 데이터를 이해할 수 있도록 해주는 계층
- 입력 또는 출력되는 데이터를 하나의 표현 형태로 변환한다.
- 필요한 번역을 수행하여 두 장치가 일관되게 전송 데이터를 서로 이해할 수 있도록 한다.
- ex. 데이터 압축, 암호화, JPEG, MPEG 포맷 등
#### 3) 세션 계층
- 통신장치 간의 상호작용을 설정하고 유지, 동기화하는 계층
- 사용자간의 세션(포트연결)이 유효한지 확인, 설정한다.
- 통신을 위한 논리적인 연결 단계
- ex. SSH, TLS 등
#### 4) 전송 계층
- 데이터 전송을 위한 논리적인 경로를 제공하는 계층
- 패킷 전송이 유효한지 확인하고 시리패한 패킷은 다시 보내는 역할을 한다.
- 신뢰성 있는 통신을 보장한다.
- 송수신 프로세스를 식별하기 위해 부가 데이터(Port)를 추가한다.
- ex. TCP, UDP 등
#### 5) 네트워크 계층
- 다중 네트워크 링크에서 패킷을 발신지로부터 목적지로 전달하는 역할을 하는 계층
- 데이터를 목적지까지 안전하고 빠르게 전달해야 한다.
- 이를 위해, 라우팅 기능을 사용한다.
- IP 주소를 정하고 최적의 경로를 제공한다 + 패킷 전달.
- ex. IP, ICMP 등
#### 6) 데이터링크 계층
- 오류없이 한 장치에서 다른 장치로 프레임을 전달하는 계층
- MAC 주소를 가지고 정확한 장치로 데이터를 통신한다.
- 오류검출과 흐름제어를 통해 데이터가 안전하게 도달하도록 한다.
- 네트워크 계층에서 정보를 받아 주소와 제어 정보를 시작과 끝에 추가한다.
- ex. Ethernet 등
#### 7) 물리 계층
- 물리적 매체를 통해 비트 흐름을 전송하기 위해 데이터를 전기신호로 바꿔 통신하는 계층
- 통신 케이블로 데이터를 주고받는다.
- ex. 전선, 전파 등
<br><br/>

### 5. TCP/IP vs OSI 7계층
#### 1) 공통점
- 계층화를 하므로써, 통신이 일어나는 과정을 단계별로 파악할 수 있다.
- 문제가 발생할 경우, 어느 계층에서 발생했는지 파악하여 유지보수하기 쉽다.

#### 2) 차이점
- TCP/IP - 4계층, OSI - 7계층
- TCP/IP - 프로토콜 기반, OSI - 역할 기반
- TCP/IP - 데이터 전송기술에 특화, OSI - 통신 전반에 대한 표준
